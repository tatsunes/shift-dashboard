# シフト人員バランス管理ツール

Googleスプレッドシートの全店舗シフト表から、新潟エリア担当院のシフトを自動で読み取り、各院の出勤人数と基本人数の過不足をダッシュボードで可視化するWebアプリケーションです。

## 機能概要

- **ダッシュボード**: 院×日付のマトリクス表示で、出勤人数の過不足を色分け（緑:適正、青:余剰、赤:不足）
- **フィルター**: 担当院の選択的表示
- **サマリー**: 当日の不足院・余剰院を一覧表示
- **設定**: 担当院リスト、基本人数、スプレッドシートIDの編集

## セットアップ手順

### 1. Google Cloud Consoleでの設定

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. 新しいプロジェクトを作成
3. **Google Sheets API**を有効化
4. **OAuth同意画面**を設定
5. **OAuth 2.0クライアントID**を作成（Webアプリケーションタイプ）
   - 承認済みJavaScript生成元: `http://localhost:8080`（開発時）とデプロイ先URL
6. **APIキー**を作成（オプション）

### 2. アプリケーション設定

1. `.env.example`を`.env`にコピー
2. `.env`に以下を設定:
   ```
   VITE_GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
   VITE_GOOGLE_API_KEY=your-api-key  # オプション
   ```

### 3. ローカルで実行

```bash
# Pythonの場合
python -m http.server 8080

# Node.jsの場合
npx serve .

# またはVS CodeのLive Extensionを使用
```

ブラウザで `http://localhost:8080` を開きます。

## 使い方

### 初回セットアップ

1. 「設定」ボタンをクリック
2. スプレッドシートIDまたはURLを入力
3. 担当院リストと基本人数を確認・編集
4. 保存して閉じる

### 日常使用

1. Googleアカウントでログイン
2. シート選択プルダウンから月次タブを選択
3. 担当院フィルターで表示する院を絞り込み
4. セルをホバーで詳細情報（ヘルプ来院院名、差分）を確認

## データソース仕様

対象となるGoogleスプレッドシートは以下の構造を想定:

- **1行目**: 日付（1〜月末まで列に並ぶ）
- **2行目**: 曜日（日・月・火・水・木・金・土）
- **スタッフ行**: 氏名の列の右に各日付ごとのシフト記号
- **グループ行**: 院名がセル結合で書かれたグループヘッダー
- **出勤人数行**: 各グループ末尾に「出勤人数」という行がある

### シフト記号の解釈

| 記号 | 解釈 |
|------|------|
| ○ | 出勤（カウント） |
| × | 休み |
| ／ | 不在 |
| A | 出勤（特別シフト、○と同等） |
| 院名 | 他院からのヘルプ来院（出勤としてカウント） |
| 背景黄色 | 受付スタッフ（カウント除外） |

## デプロイ

### Vercelへのデプロイ

```bash
# Vercel CLIをインストール
npm i -g vercel

# デプロイ
vercel --prod
```

またはGitHubリポジトリをVercelに連携。

### 静的ホスティング

ビルド不要。`index.html`, `styles.css`, `*.js` を任意の静的ホスティングにアップロード。

## 技術スタック

- **フロントエンド**: Vanilla JavaScript (ES6+)
- **スタイル**: CSS3
- **API**: Google Sheets API v4
- **認証**: Google OAuth 2.0

## 注意事項

- **認証**: Google OAuthを使用。閲覧権限のあるスプレッドシートのみアクセス可能
- **レートリミット**: Google Sheets APIは100リクエスト/100秒の制限あり
- **セキュリティ**: スプレッドシートIDは閲覧権限のあるユーザーのみアクセス可能
